# Godot Web Integration And itch.io Upload

This website can host the Godot build directly, and itch.io can host the same exported build as an HTML game.

## Recommended Flow

1. Export the Godot project for Web.
2. Test the export inside the Berong SafeScape website.
3. Zip the same export.
4. Upload that zip to itch.io as an HTML game.
5. Compare behavior between the site embed and the itch.io embed.

## 1. Export The Game From Godot

In Godot:

1. Open the project.
2. Go to Project > Export.
3. Add a Web export preset.
4. Use the Compatibility renderer for web builds.
5. Export with the filename set to `index.html`.

Important notes:

- Godot 4 web builds require WebGL 2.
- Prefer the default single-threaded web export for itch.io compatibility.
- Avoid renaming exported files after export. Godot expects the files to match the original export naming.
- If you later enable threads, you will need cross-origin isolation support. That complicates hosting and is not the best first target for itch.io.

Expected export output usually includes:

- `index.html`
- a matching `.js` loader file
- a `.wasm` file
- a `.pck` file
- splash or icon assets generated by the export

## 2. Test The Build In This Website

The website already has a route prepared for the game:

- `/emergency-dispatch`

Copy the exported Godot web build into:

- `berong-safescape-main/public/godot/emergency-dispatch/`

The final structure should look like this:

```text
berong-safescape-main/
  public/
    godot/
      emergency-dispatch/
        index.html
        game.js
        game.wasm
        game.pck
        ...other exported files
```

Then run the website and open:

- `http://localhost:3000/emergency-dispatch`

What to verify:

- The game loads without 404 errors.
- Audio starts after user interaction.
- Mouse and keyboard input behave correctly.
- Fullscreen behavior works.
- The canvas scales correctly on desktop and smaller screens.
- There are no path or case-sensitivity errors in the browser console.

## 3. Upload The Same Build To itch.io

itch.io HTML upload requirements:

- Upload a `.zip` file, not `.rar` or `.7z`.
- `index.html` must be at the root of the zip.
- All required files must be inside the zip.
- Paths must stay relative.
- File names are case-sensitive.

Steps:

1. Select all files inside the exported Godot web folder.
2. Zip those files so `index.html` is at the root of the archive.
3. Go to `https://itch.io/game/new`.
4. Create a new project.
5. Set Kind of Game to HTML.
6. Upload the zip.
7. In Embed options, either:
   - use Embed in page and set a fixed viewport size, or
   - use Click to launch in fullscreen.
8. Enable the fullscreen button in itch if needed.
9. Mark the game mobile-friendly only if your web build actually works on mobile browsers.

## 4. Suggested itch.io Settings

Good starting settings for this project:

- Launch mode: Embed in page
- Width: 1280
- Height: 720
- Fullscreen button: enabled
- Click to play: enabled
- Mobile friendly: only enable after testing on a phone

## 5. Common Issues To Watch For

### Threads and SharedArrayBuffer

If you export with threads enabled, the host must support cross-origin isolation headers. That is extra deployment work and can break embeds.

For first deployment testing, use the single-threaded web export.

### Absolute Paths

Do not use absolute asset paths that begin with `/` inside custom HTML or JavaScript. itch.io hosts the game in a subdirectory.

### Case Sensitivity

Windows may let mismatched filenames work locally. itch.io will not. `Map.png` and `map.png` are different.

### Compression and Large Files

itch.io accepts HTML builds, but very large web builds load poorly. Keep the exported build lean if possible.

### Persistence In iframes

If the game depends on browser persistence through `user://`, behavior can differ when embedded in an iframe. Test saved data explicitly.

## 6. Best Integration Strategy For This Project

Use both deployment targets:

- Website embed for your main platform integration and controlled testing.
- itch.io for public browser testing, quick distribution, and external validation.

That means the website page is your internal integration test, and itch.io is your external packaging test.
